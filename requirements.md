
	
# Group 4 Requirements Questions and Answers 
 
 
## Account features and Security 
 
**Two factor authentication**

    Because of recent, high profile data breaches, our client, FDM have decided that account security is now a top
    priority. Your system must implement 2-factor authentication for all user logins
 
**Does the system require the users to give their email address to create an account?** 
  
    The learners need to input their name email and academic level in order to register  
 
**Will the password require special characters? What should the length between?** 
 
    The password should be 8 characters long and include at least one letter and one number 
 
**Do the users require a profile page?** 
 
    The profile page should be kept basic. The user should be able to see their badges and courses completed.  
     
**What does it mean to be a lecturer?** 
 
    The lecturer must be verified and have to go through certain checks, such as CRB check. 
 
**Who will be the administrators of the website?**  
 
    The business are the administrators. 
 
**Can learners become lecturers?** 
 
    The users have to create a new account and go through the process of becoming a lecture. (As above) 
   
**Does the FAQ page require logged in users?** 
 
    FAQs should be visible to unregistered users.  
    Registered users are able to see the discussion page as well as the FAQ page. 
 
**Do users have to be logged in to see course content and or modules within the course?** 
 
    Users must be logged in to see the course content.  
    Each course should have a short demo available to unregistered users.     
 
**What can a non-user view on the first page of the website?** 
 
    Each course should have a short demo available to unregistered users. 
 
**Do you have to be a certain age to sign up?** 
 
    The learners should be over 16 years old  
 
**Can multiple people be logged in from the same network?** 
 
    No, but if they have multiple user accounts then they can. 
 
**Are there going to be more than 2 account levels (e.g. premium features)?** 
     
    No 
 
**If an admin is forced to suspend a learner account how long would the suspension last.** 
 
    It is up to the admin. 
 
**If an admin is forced to suspend a lecturer account how long will the suspension last.** 
 
    It is up to the admin. 
 
**For what reasons would you have to suspend an account** 
 
    For being inappropriate. 
 
**Is there a limit to how many reports learners and lecturers can send to administrators?**  
 
    There are no restraints. 
 
 
**Will students be allowed to leave/quit a course?** 
 
    No. A user will be shown an incomplete message if they are not completing courses. 
 
**Are there any transactions (e.g. premium courses)?** 
 
    No 
 
**What can learners upload to the system?** 
 
    Learners can only upload comments and their assignments. Assignments will be uploaded on the course page. 
 
**Does the service have to be paid for? If yes then what pay service? PayPal?** 
 
    No paid service, yet.

##Courses 
 
**Will there be a display of the top courses based on learner rate and reviews?** 
 
    This is not mandatory but can be added as an extra feature. 
 
**Is there a limit to the amount of students allowed on each course?** 
 
    The amount of courses submitted by a lecturer and enrolled by a student is unlimited. 
     
**What assignment formats should be available for lecturers? coursework? tests?** 
 
    Not all courses will have assignments.  
    Those that have assignments will be learning checks and coursework which the user will upload. 
 
**Should there be a minimum amount of students for a course to run? E.g. at least 5 students.** 
 
    There is no minimum amount of students required. 
**How can a user find a course?** 
 
    The users can search for courses using a search bar and by filtering the results 
 
**Does a course have to be approved by an administrator before students are allowed to enroll?** 
 
    Yes 
 
**How will a lecturer add content to the course? I.e. will it be a form consisting of input boxes/radio buttons with different options?** 
 
    The lecturer will have to go to an upload page where they upload the video. 
    They have to add Title name. 
    Which course the video belongs to. 
    A description of the video. 
    Any extras 
 
**Are some courses co-dependent? I.e. must a student have completed one course before they are able to understand the content of another course?** 
 
    No 
 
**If students are unhappy with their grade for a particular assignment, are they allowed to re-sit either the course or that particular assignment so as to improve their grade?**  
 
    Users can only re-sit a course once. It depends on the accreditation. 
    If the course is accredited then it can only be re-sit once. 
 
 
**Should suggestions for what course to choose next appear after a learner completes a course** 
 
    Yes, there should be suggestions so users can learn something related to their course. 
 
**Should we add the difficulty of courses?** 
 
    Yes 
 
**Should there be suggestions for what the user should learn next?** 
 
    Yes 
     
**Should the learner receive a certificate if they complete a course?** 
 
    The learner should receive badges for completing a course to a specific level.  
    If they achieve over 50% of the assessments then they should receive a badge which 
    can be viewed on their profile page. 
 
**How should the course be reviewed? Using a star scheme?** 
 
    Learners can write reviews and leave feedback on courses using a five star system. 
    There should be a five star rating along with a comment box. 
 
**Do you require a badge to proceed to the next course?** 
 
    No. 
 
**Is there a limit to how many courses a lecturer can create?** 
 
    The amount of courses submitted by a lecturer and enrolled by a student is unlimited.   
     
**Is there a limit to how many courses a learner can take?** 
 
    The system should give a warning to the learner if five or more courses that they are registered  
    haven't been completed when they try to start another one.
